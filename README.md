# CryptoZombies - Zombie Factory 🧟‍♂️

First smart contract, developed as an assignment for the **Blockchain Based Systems** course in the **Computer Science (English)** program at **Tbilisi State University**.

The contract represents a "Zombie Factory" built during Lesson 1 of the [CryptoZombies](https://cryptozombies.io/) interactive tutorial. It takes a user-inputted name and generates a unique zombie with a pseudo-randomly generated 16-digit DNA sequence.

## Concepts Covered 🧠
Throughout the development of this smart contract, I learned and applied the following fundamental Solidity principles:
* **State Variables & Integers:** Storing data permanently on the Ethereum blockchain (`uint`).
* **Math Operations:** Using mathematical operations and the modulo operator (`%`) to maintain a strict 16-digit DNA format.
* **Structs & Arrays:** Creating custom, complex data types (`struct Zombie`) and storing them in dynamic public arrays.
* **Functions:** Defining `public` and `private` functions, and utilizing the `view` modifier to optimize gas costs.
* **Keccak256 Hashing:** Generating a pseudo-random hash from a string and converting it into an integer via Typecasting.
* **Events:** Emitting signals from the smart contract (`emit`) to communicate state changes to the front-end application.
